政府採購法互動教學平台

Gov Procurement Law Tutor

以 Google 帳號登入的交談式教學網站，在已匯入之法規／函釋與題庫知識庫範圍內， 透過 RAG 檢索與 AI 生成，提供有依據的採購法規問答服務。

專案簡介投影片 · 2026

專案背景與動機

  • 政府採購法及相關子法、行政規則、工程會函釋條文繁多、位階複雜，實務人員與學習者不易快速找到正確依據。
  • 一般聊天機器人容易幻覺捏造法條，不適合作為法規諮詢工具。
  • 需要一個限定知識庫範圍、可追蹤提問紀錄、並區分管理者與一般使用者的教學平台。
  • 參考 NotebookLM 等「文件導向問答」思路，但聚焦於台灣政府採購法規領域。

專案目標

🎯 教學導向

  • 提供情境模板與常見問題範例
  • 引導使用者以正確方式提問
  • 回答附帶參考法規來源

📚 知識庫限定

  • 僅引用已匯入之法規全文與函釋
  • 搭配題庫擴展檢索與加權
  • 無依據時明確告知，不臆測作答

🔐 身分與紀錄

  • Google OAuth 登入
  • 區分管理者／一般使用者
  • 保存個人提問與回答紀錄

🚀 可部署維運

  • Next.js + PostgreSQL 架構
  • 支援 Vercel + Neon 雲端部署
  • 管理者可更新知識庫

知識庫與 RAG 回答流程

  1. 語料匯入：從 data/corpus/ Markdown 或全國法規資料庫 API 下載法規全文
  2. 分塊與向量化：條文切分為 DocChunk，選用 OpenAI 產生 embedding
  3. 題庫輔助：QuestionBankItem 提供關鍵詞擴展與相關法規 slug 加權
  4. 混合檢索：語意相似度 + 關鍵字比對；無匹配時不回傳無關片段
  5. 生成回答：將檢索片段作為 context，要求模型僅依片段作答並標註來源
  6. 降級機制：OpenAI 不可用時，改以關鍵字摘錄 fallback 回覆

法規涵蓋範圍（節錄）

  • 政府採購法、施行細則
  • 未達公告金額採購招標／監辦辦法
  • 最有利標評選辦法、採購評選委員會組織準則
  • 投標廠商資格與特殊或巨額採購認定標準
  • 採購契約要項、採購申訴審議規則
  • 各類型採購錯誤行為態樣
  • 公共工程招標文件公開閱覽制度實施要點
  • 統包作業須知、國際競圖注意事項
  • 專業／技術／資訊服務評選及計費辦法
  • 工程會採購金額門檻、保證金作業規定等
  • 資源回收再利用法（相關採購）
  • 評選委員會專家學者名單資料庫相關要點

完整清單見網站「法規／函釋清單」頁面，依法律 → 法規命令 → 行政規則 → 函釋位階排序。

歷史沿革

第一階段｜專案啟動
确立「交談式政府採購法教學網站」需求，建立 Next.js + Prisma + PostgreSQL 基礎架構。
第二階段｜身分與權限
整合 Google OAuth 登入（NextAuth v5），實作 ADMIN／USER 角色與提問紀錄儲存。
第三階段｜知識庫建置
匯入政府採購法相關法規清單與 corpus 文本，建立法規位階排序與清單頁面。
第四階段｜RAG 問答
實作條文分塊、embedding 語意檢索、OpenAI 生成回答與來源引用機制。
第五階段｜語料擴充
串接全國法規資料庫 API、工程會要點匯入、PDF 題庫解析與 question-bank 整合。
第六階段｜品質與體驗
檢索精準度優化、OAuth／驗證修正、UI 區塊配色、等待動畫、LOGO 與部署文件完善。
